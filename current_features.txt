BodyPilot — Feature Implementation Audit
=========================================

Last refreshed: 2026-06-10. Compare against feature_list.txt for the full
product vision and phase_plan.txt for in-flight work.

Legend
------
- ✅ Implemented (real, working logic — not a placeholder UI)
- ⚠️ Partial / demo / behind an API key or external prerequisite
- ❌ Not implemented (no code or only an inert button)

How this audit was produced
---------------------------
Grepped the codebase for keywords matching each feature, then opened the
candidate files to confirm the implementation is real (functions/screens/
stores wired in, not just type definitions or stubs). Citations point to
the primary entry point or library file.

## Summary

Roughly **62 ✅ implemented · 15 ⚠️ partial · 31 ❌ not implemented**
across the 12 sections above.

**Strongest areas:** workout intelligence (deload, progression, splits,
VO2, HR zones, GPS cardio), nutrition (AI photo / barcode / receipt
scanners, NL log with voice, IIFYM, adaptive calories, metabolic
adaptation, gut score), gamification, social, Apple HealthKit, analytics
+ exports, and now iOS widgets.

**Biggest remaining gaps:** non-Apple wearable / device integrations,
camera-based form coaching and rep counting, virtual-race-style social,
education / academy content, monetization (StoreKit transactions),
accessibility (a11y labels), and internationalization.
