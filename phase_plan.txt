## Phase A: Foundation Upgrades (do first — unblocks everything)

**These are infrastructure changes that multiple future features depend on.**

1. **Offline-First Persistence** — Add Zustand persist middleware with AsyncStorage so all stores survive app restart. Add `@react-native-community/netinfo` for connectivity detection + offline banner + retry queue for Supabase writes
2. **Expanded Exercise Database** — Replace the 10 hardcoded exercises with 200+ seeded into Supabase (with muscle groups, equipment, instructions, tips). This unblocks workout builder, split optimizer, warm-up generator
3. **External Food API Integration** — Wire up OpenFoodFacts/USDA FoodData Central API in food-search.tsx to replace the 30 hardcoded items. This unblocks barcode scanner, natural language logging, and accurate micros
4. **Real Barcode Scanning** — Integrate `expo-camera` barcode mode → look up scanned code via OpenFoodFacts API
5. **Real AI Food Scanner** — Wire the existing `analyzeFoodPhoto()` in openai.ts to the ai-scanner.tsx screen using `expo-image-picker`
6. **Data Export** — Implement actual CSV/JSON export of all user data (workouts, nutrition, measurements, goals) in settings.tsx
7. **Theme System** — Make the dark/light/system toggle actually apply by wrapping Colors in a ThemeContext
8. **Accessibility Pass** — Add `accessibilityLabel`, `accessibilityRole`, `accessibilityHint` to all interactive elements, buttons, inputs, cards across every screen

**Estimated effort: ~2-3 weeks**

## Phase B: Workout Intelligence

9. **Auto-Progression Engine** — After each workout, compare to previous session → suggest weight/rep increases next time. Store progression rules per exercise
10. **Superset/Circuit/EMOM/AMRAP Modes** — Add `workout_type` field to active workout, build timer modes (EMOM countdown, AMRAP clock), group exercises into supersets
11. **Warm-Up & Cool-Down Generator** — AI generates dynamic warm-up based on today's workout muscles + static cool-down. New screen `/workout/warmup`
12. **Training Split Optimizer** — AI analyzes recovery data + workout history → suggests optimal weekly split. New screen or section in AI chat
13. **Deload Detection** — Track rolling 4-week volume. When fatigue ratio exceeds threshold, prompt deload week with reduced program
14. **Strength Standards** — Compare 1RM estimates to population percentiles by bodyweight/age/sex (use public strength standards data)
15. **GPS Cardio Tracking** — New `/workout/cardio` screen using `expo-location` for run/cycle tracking with pace, distance, elevation, map replay
16. **VO2 Max Estimation** — Calculate from cardio data (Cooper test formula or HR-based estimation)
17. **Heart Rate Zone Training** — If BLE HR monitor connected, show real-time zone overlay during cardio

**Estimated effort: ~3-4 weeks**
